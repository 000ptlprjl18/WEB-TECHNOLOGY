JWT – Short Notes

1. What is JWT?

JWT (JSON Web Token) is an open standard (RFC 7519) used to securely transmit information between a client and a server as a JSON object. It is commonly used for authentication and authorization in modern web applications.

2. Structure of a JWT

A JWT consists of three parts separated by dots (.):

Header.Payload.Signature

Example:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
eyJpZCI6IjEyMyIsIm5hbWUiOiJWYWliaGF2In0.
hSDSDke4JSd-9Jsd0ad4J9da2xP3a

a) Header

Contains metadata about the token such as type and algorithm used for signing.

{
  "alg": "HS256",
  "typ": "JWT"
}

b) Payload

Contains claims or data. These can be predefined or custom claims.

{
  "userId": "123",
  "role": "admin",
  "exp": 1736543210
}

c) Signature

Created by encoding the header and payload using Base64, then signing with a secret key.

HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)

3. How JWT Works

  1. User logs in with credentials.
  2. Server verifies credentials and generates a JWT.
  3. JWT is sent to the client and stored (usually in localStorage or cookies).
  4. Client sends JWT with each request (in headers as Authorization: Bearer <token>).
  5. Server verifies token before granting access to protected routes.

4. Advantages of JWT

  • Stateless – no need to store sessions on the server.
  • Compact – easily transmitted via URLs, headers, or cookies.
  • Secure – signature ensures data integrity.
  • Cross-platform – can be used across different languages and systems.

5. Common Use Cases

  • User authentication and authorization.
  • Secure API communication.
  • Single Sign-On (SSO).

6. Example in Node.js

const jwt = require('jsonwebtoken');

const token = jwt.sign(
  { email: 'user@example.com' },
  'secret_key',
  { expiresIn: '1h' }
);

const verified = jwt.verify(token, 'secret_key');
console.log(verified);

7. Drawbacks of JWT

  • Cannot be revoked easily once issued.
  • Larger size compared to simple session IDs.
  • Must be securely stored to prevent token theft.

8. JWT vs Session

JWT Session
Stateless (no server storage) Stateful (stored on server)
Token stored in client Session ID stored in cookie
Faster and scalable May slow down with many users
Cannot be easily invalidated Can be destroyed on logout
